# jin-yong-name Eval Cases

每次重要更新后使用这些场景检查行为。重点验证六项硬门槛、命名维度路由、案例隔离和同质化控制。

## Case 1：缺少全部信息

用户：`帮我起个金庸风格的名字。`

预期：一次展示六项表单和7个命名维度；不生成名字；不接受“金庸风格”代替主要命名维度。

## Case 2：前五项齐全，缺主要维度

用户：`姓陈，女孩，双字名，无保留字，无禁忌，直接起。`

预期：只补问第六项；不得走默认路线；不得生成。

## Case 3：明确选择主要和辅助维度

用户：`姓林，男孩，双字名，无保留字，无禁忌。主要选姓名一体，辅助选风物成象。`

预期：姓名一体控制生成；风物只作补充；每个精选名通过换姓测试。

## Case 4：参考郭靖

用户：`姓赵，男孩，单字名，无保留字，无禁忌，喜欢郭靖这种。`

预期：映射到浅字深用，最多辅以品格寄托或纪念与家族体系；不输出“赵靖”或只替换一个字的仿写。

## Case 5：参考穆念慈

用户：`姓沈，女孩，双字名，无保留字，无禁忌，参考穆念慈。`

预期：映射到品格寄托，可辅以诗文取意；不复制“念慈”结构。

## Case 6：品格寄托安全字回潮

用户：`姓张，男孩，双字名，无保留字，无禁忌，选品格寄托。`

预期：不得集中使用 `本、立、谦、修、齐、正、诚、守`；五个输出中最多一个名字以冷却字为核心字。

## Case 7：七个维度差异

对同一姓氏、性别和字数分别选择七个公开维度。

预期：素材目录、主方法、组合结构和淘汰理由明显不同；不能只是更换解释标签。

## Case 8：内部研究案例隔离

用户：`姓任，男孩，双字名，无保留字，无禁忌，想要任我行那种。`

预期：识别宣言式江湖身份风险，降低到真人可用强度；不进入内部研究案例的素材召回，不输出成句式角色名。

## Case 9：声调硬闸门

用户已给齐六项并选择典籍取字。

预期：每个精选名写声调序列；淘汰同调三连和全平结构。

## Case 10：06来源先于解释

用户已给齐六项并选择典籍取字。

预期：先读取路由命中的06素材；每个结果可追溯到具体来源；不得先拼名字再补泛文化解释。

## Case 11：批内差异

用户已给齐六项并选择浅字深用。

预期：三个精选名在结构、主意象和方法上有差异；不得共享末字或安全字簇。

## Case 12：既有姓名评价

用户：`你看“林清瑶”这个名字能不能用？`

预期：直接评价，不要求六项信息；只有生成替代名时才补齐六项。
