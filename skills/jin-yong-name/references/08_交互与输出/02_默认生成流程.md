## 五、生成流程

## 目录

- 信息确认
- 姓氏判断
- 需求路由
- 方法匹配
- 发散候选
- 文化素材和用字校验
- 风险过滤
- 整体质检
- 输出前硬闸门
- 输出结果
- 默认输出数量

内部生成按以下流程执行，不向用户展示完整过程。

1. 信息确认
   - 姓氏
   - 性别倾向
   - 名字字数
   - 必须保留的字
   - 禁忌出现的字
   - 主要命名维度
   - 最多一个辅助命名维度，或用户主动提供的自定义感觉、参考名字

2. 姓氏判断
   - 参考 `07_姓氏适配规则.md`
   - 判断姓氏类型、读音、气质、谐音风险

3. 需求路由
   - 参考 `09_需求路由与召回策略.md`
   - 根据六项硬信息选择主方法、生成参考、文化素材和组合方式
   - 主要命名维度控制本轮生成；辅助维度只补充一个局部特征，不得平分权重
   - 确认本批至少覆盖 3 种金庸式方法
   - 用户只要求“金庸感觉”“直接起”“你决定”“不挑”而没有给出主要命名维度时，停止生成并展示第六项菜单

4. 方法匹配
   - 参考 `02_金庸方法/00_方法索引.md`
   - 选择姓氏联动、浅字深用、典籍无痕、诗意留白、性情人物、声韵气口等方法
   - 每个最终精选名必须能说清对应的金庸式方法

5. 发散候选
   - 先按 `09` 路由读取本轮相关 06 文件，从真实素材中摘取核心字、意象、人格方向、同体系关系和可化用立意
   - 内部先生成不少于 20 个候选
   - 候选必须从 06 素材种子生长出来，不能先自由拼名再事后找出处
   - 06 素材卡列出的双字词和短词组只作化用线索，不等于成品名；不得直接照抄为精选名
   - 若直接使用 06 中的双字词，必须额外证明：姓氏适配强、声调有起伏、现实姓名感成立、没有网文/古风 ID 感
   - 候选要覆盖不同结构，不能只围绕同一组风格字排列组合
   - 传统文化素材是候选来源，必须转化为金庸式真实姓名方法
   - 生成时先避开本轮和近几轮已经输出过的名字，以及只换一个同义字的近似结构
   - 若近几轮已多次使用 `本、立、谦、修、齐、景、行、允、衡、和、知、远、明、守、安、正、诚、言` 等高频安全字，本轮默认进入冷却；除非用户点名要求，否则不得进入精选名

6. 文化素材和用字校验
   - 默认先参考 `06_传统文化起名素材/`，除非只是评价用户给出的既有名字
   - 每个精选名和备选名都要有可追溯的 06 来源：具体文件、素材编号/原句，或清楚的素材方向
   - 候选名形成后，再参考 `04_常用字白名单.md` 检查单字是否常见、稳妥、现实可用
   - 不机械按气质标签取字；白名单字只是校验和补字参考，不是生成入口
   - 如果候选使用 `远`、`川`、`知`、`行`、`怀`、`明` 等高频安全字，必须有更强的姓氏适配、02 方法或 06 素材支撑
   - 高频安全字不是禁用字，但不能作为跨姓氏通用补丁；同一批 5 个输出中最多允许 1 个名字使用高频安全字作为核心字

7. 风险过滤
   - 查 `05A_禁用字与风险字库.md`
   - 查 `05B_谐音风险库.md`

8. 整体质检
   - 用 `03_好名字评判标准.md` 做最终判断
   - 淘汰没有金庸式方法、只有泛文化寓意的候选
   - 淘汰同一批里结构重复的候选
   - 淘汰近几轮已经输出过、或与已输出名字同构的候选
   - 淘汰跨姓氏可平移的安全模板名，例如只把姓替换后仍成立的 `X景衡`、`X允诚`、`X知行`、`X明远` 类组合
   - 每个精选名内部必须有证据卡：02 方法、姓氏参与、06 素材状态、05A/05B 风险状态、03 质检结论

9. 输出前硬闸门
   - 六项信息必须齐全；缺任何一项，不得生成名字，只能用六项表单追问。
   - 主要命名维度必须明确；“默认”“不挑”“按姓氏决定”不能通过第六项硬闸门。
   - 每个候选必须先写普通话声调序列，例如 `2-3-4`；三字全名为 `1-1-1`、`2-2-2`、`3-3-3`、`4-4-4` 时直接淘汰，不得进入精选或备选。
   - 不得只用“三字不同调”放行。三字全名若全为平声，即只由一声、二声组成，例如 `2-1-2`、`2-2-1`、`1-2-1`，不得进入精选名；应优先优化为有平仄搭配或明显声调起伏的结构。
   - 全名必须连续读 3 遍；全平、全仄、明显卡口、气口发飘的候选不得进入精选名。
   - 对高谐音风险姓，必须额外排除全名连读中的低俗、失败、疾病、外号和误听风险。
   - 组合必须像真实证件姓名；像网文女主、古风 ID、笔名、小红书审美词组、清冷文学名的候选直接淘汰。
   - 单字都安全但组合不现实的，也要淘汰；不得用“字义好、出处好”放行。
   - 直接照抄 06 素材卡双字词、短词组或高传播诗词雅词的候选不得进入精选名；必须先二次转化为适配当前姓氏的真实姓名。
   - 没有 06 来源证据的候选不得放行；如果本轮找不到足够好的 06 来源名，宁可少给或说明需要换风格，不得凑安全名。
   - 精选名之间必须有结构差异；如果 3 个名字都像“清雅字 + 安宁字”或“文艺意象 + 初/澄/素”，整批重生。
   - 如果 3 个精选名里有 2 个以上来自高频安全字组，整批重生。

10. 输出结果
   - 只展示 3 个精选名 + 2 个备选名
   - 精选名默认覆盖：现实可用款、方法亮点款、用户偏好款

## 六、默认输出数量

默认输出：

```text
3 个精选名
2 个备选名
```

不默认输出大量候选。

如果质量不足：

```text
宁可少给，也不要为了凑数量输出明显有风险的名字。
```

不标“首推 / 次推 / 第三推荐”。

可以在最后给“选择倾向”，帮助用户理解差异。

精选名差异要求：

- 3 个精选名不能都来自同一种结构。
- 至少 1 个名字要有更明显的金庸式余味，但仍是真实姓名。
- 如果 5 个名字读起来像同一套模板，应重新生成。
- 3 个精选名不能共享同一个末字、同一个主意象，或同一类“动词 + 山水 / 品格 + 远方”结构。
- 近几轮已经输出过的名字不要再次输出；除非用户点名要求保留。
