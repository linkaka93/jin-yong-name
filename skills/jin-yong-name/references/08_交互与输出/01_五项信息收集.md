# 六项信息收集与命名维度选择

## 一、交互原则

起名前一次性收集六项硬信息。第六项是主要命名维度，缺少时不得生成名字。

不再按男孩、女孩、中性展示不同风格菜单；不提供“默认”“不挑”“按姓氏决定”。用户已经主动给出明确取法、感觉或参考名字时，视为第六项已完成，内部映射到一个主要命名维度，必要时增加一个辅助维度。

## 二、六项硬信息

```text
请先补充这 6 项，我再为你起名：

1. 姓氏：
2. 性别：男孩 / 女孩 / 中性
3. 名字字数：单字名 / 双字名 / 都可以
4. 必须保留的字：没有填“无”
5. 禁忌出现的字：没有填“无”
6. 你希望名字通过哪种方式成立？

   1. 品格寄托｜杨铁心、穆念慈、刘正风
   2. 典籍取字｜程灵素、张无忌、宁中则
   3. 诗文取意｜木婉清、周芷若、李秋水
   4. 姓名一体｜风清扬、木婉清、温仪
   5. 风物成象｜霍青桐、阮星竹、邓百川
   6. 浅字深用｜郭靖、刘正风、萧中慧
   7. 纪念与家族体系｜郭靖 / 杨康、武敦儒 / 武修文、袁承志
   8. 自定义｜写下喜欢的感觉或参考名字

请选择一个主要方向；如有需要，可再补充一个辅助方向。
```

六项说明：

- 姓氏必须参与全名判断。
- 性别只控制用字倾向，不做刻板限制。
- 必须保留的字仍需经过风险检查。
- 禁忌字严格避开。
- 主要命名维度决定本轮素材召回、组合方式和淘汰标准。
- 辅助维度最多一个，只作补充，不能与主要维度平分权重。

## 三、七个公开命名维度

### 1. 品格寄托

用名字表达做人方向、心性和亲长期许。重点是含蓄、有分量，不写成训话；不得集中堆叠 `正、修、谦、诚、守`。

### 2. 典籍取字

从典籍篇名、历史人物、原句或思想体系中提取核心字。名字表面自然，出处藏在背后。

### 3. 诗文取意

从诗文中取意象、语气或留白，不直接照搬高传播雅句，不生成网红古风名。

### 4. 姓名一体

让姓氏参与意义、气质或画面。换成其他姓氏后，整体效果应明显下降。

### 5. 风物成象

借草木、山水、地理、时序或光影形成画面。限制仙侠感、角色感和常见风物字堆叠。

### 6. 浅字深用

使用容易读写的常见字，通过组合、结构和含义形成分量。重点解决大姓普通和安全字模板问题。

### 7. 纪念与家族体系

通过历史纪念、家族期许、同辈关系或成对结构建立名字。单人起名时转化为含蓄的家庭寄意；兄弟姐妹起名时允许体系化设计。

## 四、自定义与参考名字

用户可以填写喜欢的感觉、参考人物名、希望接近的命名方法或不希望出现的感觉。

处理规则：

1. 映射为一个主要命名维度，最多增加一个辅助维度。
2. 用户给出参考名时，拆解其命名机制，不复制原名，不做只换一个字的仿写。
3. 用户给出多个要求时，保留最核心的一个主要求和一个辅助要求。
4. “仙气、江湖、惊艳、像小说人物”等要求自动降为真人姓名可用强度。
5. 用户只说“金庸感觉”“直接起”“你决定”“不挑”时，第六项仍未完成，必须展示本菜单请其选择。

## 五、通用硬闸门

声韵气口、姓氏适配、现实可用、文化来源和风险过滤属于所有路线的通用检查，不作为用户选项。
