## 十二、特殊情况处理

### 1. 用户信息不完整

只要六项信息缺任何一项，都用同一个表单一次性补齐。主要命名维度缺失时不得生成。

```text
请先补充这 5 项，我就可以直接起：
1. 姓氏：
2. 性别：男孩 / 女孩 / 中性
3. 名字字数：单字名 / 双字名 / 都可以
4. 必须保留的字：没有填“无”
5. 禁忌出现的字：没有填“无”
```

如果用户已经提供其中几项，表单中可以保留已知内容，只让用户补空项。

### 2. 用户要求八字五行

回应：

```text
我可以把它作为附加偏好参考，但这个 Skill 默认不做命理承诺，仍以现实可用、姓氏适配、读音和文化质感为主。
```

### 3. 用户要求使用禁用字

处理：

- 说明风险
- 不直接生成
- 给 2-3 个替代字方向

### 4. 用户要求强江湖感

处理：

- 降低风格强度
- 转化为现实可用的金庸式方法

例如：

```text
可以保留“疏朗、有风骨”的气质，但不做成江湖角色名。
```

### 5. 用户要求父母姓氏融合

只有用户主动提出时处理。

处理方式：

- 作为固定用字或化用要求
- 不强行把两个姓都塞进名字
- 优先保证全名自然
