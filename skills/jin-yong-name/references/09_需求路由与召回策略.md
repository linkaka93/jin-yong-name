# 09_命名维度路由与召回策略

## 一、定位

本文件把用户选择的主要命名维度转成实际生成路线。主要命名维度必须改变：主方法、素材召回、组合方式和淘汰标准，不能只改变结果说明。

统一原则：

- 六项硬信息齐全后才能生成。
- 用户主选一个维度，可补充一个辅助维度。
- 辅助维度只能补充局部特征，不得与主要维度平分权重。
- 声韵气口、姓氏适配、现实可用、06 来源、05A/05B 风险过滤和03整体质检是通用硬闸门。
- 40组人物案例只用于理解方法，不是名字素材库；不得复用原名或做单字替换。

## 二、七个公开维度

### 1. 品格寄托

代表：杨铁心、穆念慈、刘正风

主方法：品格寄托 + 浅字深用。

优先素材：

- `06F_传统人文精神`
- 按具体立意少量读取 `06A`
- 需要含蓄表达时补充 `06B`

组合要求：

- 把品格写成可感知的人物性情，不写成训话或口号。
- 候选须覆盖不同人格方向，不能全部围绕端正、自律、谦逊。
- `正、修、谦、诚、守、本、立、齐`进入冷却，除非有强姓氏适配和明确来源。

### 2. 典籍取字

代表：程灵素、张无忌、宁中则

主方法：典籍无痕 + 浅字深用。

优先素材：

- 根据用户要求定向读取 `06A`、`06B`、`06D`、`06E` 或 `06F`
- 不默认偏向儒家素材

组合要求：

- 从同一篇章、同一思想体系或互相呼应的来源取核心字和立意。
- 不直接照搬篇名、成语、原句或素材卡双字词。
- 出处深，名字表面仍要自然易用。

### 3. 诗文取意

代表：木婉清、周芷若、李秋水

主方法：诗词化用 + 留白克制。

优先素材：

- `06C_诗骚`
- `06C_汉魏`
- `06C_唐诗`
- `06C_宋词`
- 强制参考 `06C_慎用避坑`

组合要求：

- 取语气、画面和立意，不取高传播雅词成品。
- 淘汰花月香露、美玉香草机械堆叠和古风ID感。
- 不因诗文来源漂亮而放宽真实姓名感。

### 4. 姓名一体

代表：风清扬、木婉清、温仪

主方法：姓氏联动 + 浅字深用。

优先素材：

- 先读 `07_姓氏适配规则.md`
- 再根据姓氏画面定向读取 `06B`、`06C`、`06D` 或 `06F`

组合要求：

- 名字顺着姓氏的字义、音色或气质生长，但不重复姓氏意象。
- 每个精选名都做“换姓测试”：换成常见姓后质感若基本不变，姓名一体不成立。
- 高意象姓避免再堆同类风物字。

### 5. 风物成象

代表：霍青桐、阮星竹、邓百川

主方法：意境风物 + 性情人物。

优先素材：

- `06B`
- `06C`
- `06D`

组合要求：

- 一名只保留一个主画面，风物必须转化为人物气质。
- 山、水、川、竹、星等常见意象不得直接拼成风景名。
- 淘汰仙侠、隐士、高人、游戏角色和强小说感。

### 6. 浅字深用

代表：郭靖、刘正风、萧中慧

主方法：浅字深用 + 声韵气口。

优先素材：

- 根据姓氏跨 `06A` 至 `06F` 定向召回
- `04`只在候选形成后做可用性校验

组合要求：

- 使用常见字，但每个候选必须有具体来源关系、姓氏适配或结构亮点。
- 不以安全字轮换代替创意。
- 单字名必须有足够承重能力；双字名要有主次关系。

### 7. 纪念与家族体系

代表：郭靖 / 杨康、武敦儒 / 武修文、袁承志

主方法：纪念寄意 + 同辈体系化 + 品格寄托。

优先素材：

- 用户真实提供的家族信息、纪念主题或同辈关系
- 与主题相符的 `06A`、`06E`、`06F`

组合要求：

- 单人起名时，把纪念和家族期许转化得含蓄，不背负过重使命。
- 成对或同辈命名要同体系、有区分，不能复制相同尾字模板。
- 未提供真实纪念对象或家族关系时，不编造家族故事。

## 三、自定义与参考名映射

用户填写感觉、参考人物名或其他真实姓名时：

1. 先判断其主要命名机制，映射到一个公开维度。
2. 只有参考名确实由两种方法共同成立时，增加一个辅助维度。
3. 不复制参考名，不替换其中一个字后输出，不借人物命运或身份滤镜。
4. 自定义要求与真人姓名边界冲突时，降低强度但保留核心意图。
5. 无法判断主维度时，展示七个维度请用户选择，不擅自使用默认路线。

常见映射：

| 用户表达 | 主要维度 | 可选辅助维度 |
|---|---|---|
| 喜欢郭靖这种 | 浅字深用 | 品格寄托或纪念与家族体系 |
| 参考穆念慈 | 品格寄托 | 诗文取意 |
| 想从经典里取 | 典籍取字 | 姓名一体 |
| 希望姓和名连起来 | 姓名一体 | 风物成象 |
| 想给兄弟姐妹一起起 | 纪念与家族体系 | 按实际主题补充一个维度 |

## 四、候选生成与差异化

1. 内部先生成不少于20个候选。
2. 至少覆盖三种金庸式基础方法，但主维度必须在最终精选中清晰可见。
3. 三个精选名不得共享同一末字、主意象或结构。
4. 每个精选名保留内部证据卡：主要维度、辅助维度、02方法、姓氏参与、06来源、05A/05B状态、03结论。
5. 案例库中的人物全名、强绑定名字和只改一个字的近似结构不得进入输出。

## 五、高频安全字冷却

重点冷却：

```text
本、立、谦、修、齐、景、行、允、衡、和、知、远、明、守、安、正、诚、言
```

执行规则：

- 高频字不是永久禁用字，但不能作为跨姓氏通用补丁。
- 同一批五个输出中，最多一个名字以冷却字为核心字。
- “品格寄托”和“浅字深用”路线不得因此放宽冷却。
- 若两个以上精选名依赖冷却字，整批重生。
- 新的字簇连续跨姓氏复用时，按同样规则加入临时冷却，不等用户再次指出。

## 六、内部研究案例隔离

`02_金庸方法/10_慎用与禁用案例.md` 中的命运反讽、江湖身份、强角色塑造和身份变化案例只用于识别风险：

- 不进入用户菜单。
- 不得进入素材召回。
- 不作为输出中的正向对标。
- 不得把人物命运、门派身份或角色强度转化成真人姓名卖点。
